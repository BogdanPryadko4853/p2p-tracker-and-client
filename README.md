# p2p file sharing system

## tracker
### Функциональные требования
1. Регистрировать пира при запуске клиента (принимать IP, порт, список раздаваемых файлов).
2. Принимать периодические сигналы (heartbeat) от пира, чтобы знать, что он ещё жив.
3. Удалять неактивных пиров (например, не присылавших heartbeat больше 2 минут).
4. Искать файлы по имени (или части имени) и возвращать список найденных файлов (их хеши, имена, размеры).
5. Возвращать список активных пиров для конкретного файла по его хешу (IP и порт для прямого соединения).
6. Обновлять список файлов пира (когда пользователь добавляет или удаляет раздачу).

### Технологический стек
1. Spring Boot 4.0.3
2. Spring Data JPA
3. PostgreSQl
4. Lombok
5. Java 17

### Модели данных
1. Peer - пир
2. FileInfo - Информация о файле

**Peer:**
1) id (UUID) — первичный ключ
2) ip (String) — IPv4 или IPv6
3) port (int) — порт для P2P-соединений
4) lastSeen (LocalDateTime) — временная метка последнего heartbeat
5) files (List<FileInfo>) — связь ManyToMany

**FileInfo:**
1) hash (String) — первичный ключ, SHA-1 хеш файла
2) name (String) — имя файла
3) size (long) — размер в байтах
4) peers (List<Peer>) — обратная связь ManyToMany

### REST API
1. `POST /api/peers/register` -  Регистрация пира
2. `POST /api/peers/{peerId}/heartbeat` - Обновляет lastSeen у пира
3. `POST /api/peers/{peerId}/files` - Заменяет текущий список файлов пира новым
4. `GET /api/files/search?query=часть_имени` - Поиск файлов
5. `GET /api/files/{hash}/peers` - Получение пиров для файла

### Сервисный слой
1. PeerService
2. FileService
3. Scheduled очистка - Каждую минуту удалять пиров, у которых lastSeen меньше текущего времени минус 2 минуты

### Дополнительные функциональные требования (скоро добавлю во втором релизе)
1) Удаление пира при graceful shutdown — если клиент явно отключается, он может отправить запрос на дерегистрацию.
2) Защита от дубликатов файлов — при регистрации файла с существующим хешем не создавать новый FileInfo, а использовать существующий.
3) Пагинация для поиска — чтобы не возвращать тысячи результатов сразу

#### Настройка запуска
